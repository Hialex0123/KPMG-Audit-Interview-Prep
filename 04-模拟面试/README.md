## 第一关：群面 · 地铁线路规划 Case

> 最高频 Case，建议重点准备。全程 55 分钟（约40分钟讨论 + 15分钟汇报）。

### 你的角色：大学校长

你代表辖区内某大学，关注学生通勤需求和教育区的人流量覆盖。

### 推荐讨论框架（可在开场主动提出）

```
① 明确约束：预算上限、人流量目标、站点数量限制
② 识别关键节点：大学、商圈、医院、政府机构、交通枢纽
③ 画草图：靠近白板的人先勾勒初步路线
④ 分工计算：人流量组 / 预算组 / 路线绘制组
⑤ 取舍优化：根据预算舍弃次要节点
⑥ 确定线路主题，准备汇报
```

### 开场发言参考（大学校长角色）

> *"我从大学方面的诉求出发：我们学校日均通勤学生约1.2万人，早晚高峰明显，目前主要依赖公交，耗时长。从学生视角，我建议核心路线覆盖'学校 — 市中心 — 就业聚集区'这条主轴。同时我想提一个线路主题方向：'青年发展线'，连接高校、创业园和就业中心，这样定位清晰，汇报时也有逻辑。大家觉得这个方向可行吗？我们先确认约束条件，然后开始分工？"*

### 推动讨论的关键句

| 情景 | 用语 |
|------|------|
| 推进分工 | *"我们现在还有30分钟，建议分三组：一组算人流，一组算预算，一组负责画图，我来协调整合，可以吗？"* |
| 整合分歧 | *"我理解公交会长担心竞争问题，我们可以在这段设换乘节点，这样既不冲突，还能提高总体覆盖率。"* |
| 时间提醒 | *"现在还有15分钟，我们需要开始整理最终方案了，请大家把各自的计算结果报一下。"* |
| 汇报总结 | *"我来总结一下我们的方案：路线主题是'青年发展线'，共X站，预计覆盖日均人流X万，总预算X亿，控制在约束范围内。"* |

### Q6：你如何理解审计？审计和财务有什么区别？

**✅ 参考回答：**

> "审计本质是**独立的第三方鉴证**。
>
> 财务人员在企业内部，职责是'记录和呈现'；审计师是外部的，职责是'核实和验证'——判断财务报表是否真实、公允地反映了企业实际状况。
>
> 从操作层面：财务更偏日常账务处理和报表生成；审计更偏抽样、分析异常、核查凭证和与客户沟通取证。
>
> 我在实习中的体会印证了这一点：风险排查时，我不只是看账面数字，而是要判断'这个数字和实际运营是否一致'——这就是审计思维的核心。"

### Q7：描述一次团队冲突，你是怎么解决的？（STAR法则）

**✅ 参考回答：**

> "**背景**：担任天文协会会长时，我们联合周边两所高校策划一次大型观星活动，对方坚持在自己学校场地举办，但那个场地光污染严重，效果很差。双方僵持了一周。
>
> **我的做法**：我先分别和双方负责人单独聊，了解各自的核心诉求。发现对方真正在意的是参与人数和品牌曝光，而不是场地本身。于是我提出折衷方案：主活动放到光污染少的地点，但宣传材料联合署名，报名通道向对方成员优先开放。
>
> **结果**：双方都接受了，当天到场超过200人，合作方的曝光目标也实现了。
>
> 那次让我意识到：冲突很多时候不是立场对立，而是诉求没有被充分表达。先听、再找交集，通常能找到双赢方案。"

### Q8：如果在工作中发现了错误，你怎么处理？

**✅ 参考回答：**

> "三步：**确认 → 上报 → 复盘**。
>
> 首先确认不是自己理解有误，把疑问整理清楚。然后及时向上反馈，不因为怕担责而拖延——审计工作里，一个小错误如果被压着没处理，可能影响整个报告结论。最后复盘原因，是流程问题还是个人失误，避免下次重复。
>
> 实习时我确实犯过错——写周报时把两组数据的口径混淆了，在最后核对时自己发现的。我立刻向负责人说明，修正后重新发出，没有延误。那次之后我养成了多加一道核对确认的习惯。"
